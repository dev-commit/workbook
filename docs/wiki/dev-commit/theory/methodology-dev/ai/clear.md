# Фреймворк CLEAR

## Информация

::: tip CLEAR

- **CLEAR** — это мнемонический фреймворк для формулирования промптов под ИИ (LLM).
- Когда важны краткость и встроенная проверка результата.

:::

## Элементы

- **C — Concise** (краткость). Промпт без лишних слов: только нужный контекст и задача.
- **L — Logical** (логичность). Чёткая структура: порядок шагов, условия, формат ответа.
- **E — Explicit** (явность). Явно указать: что нужно сделать, в каком формате, какие ограничения и критерии качества.
- **A — Adaptive** (адаптивность). Промпт можно легко менять под разные сценарии (другой тон, длина, язык, роль).
- **R — Reflective** (рефлексивность). Добавить проверку результата: «проверь, что…», «если X — сделай Y», или запрос на самопроверку модели.#

## Пример запрос

Задача: «Добавь экран списка элементов с поиском и пагинацией.»

**C — Concise (кратко)**

- Одна цель в одном запросе:
- «Реализуй экран: список элементов, поле поиска и пагинация (кнопки «Назад» / «Вперёд»).»

**L — Logical (логично)**

- Порядок и структура:
- «1. Загрузка списка с API при монтировании. 2. Поиск — запрос с параметром q при вводе (debounce 300 ms). 3. Пагинация — запрос с page и pageSize. 4. Состояния: загрузка, пустой результат, ошибка — отдельная разметка для каждого.»

**E — Explicit (явно)**

- Жёсткие условия:
- «Список — таблица или карточки (уточни один вариант). Поиск — одна строка над списком. Не больше 20 элементов на страницу. Ошибки показывать тостом/баннером, не только в консоли.»

**A — Adaptive (адаптивно)**

- Разные сценарии:
- «На мобилке — карточки в одну колонку, на десктопе — таблица. Если API вернул пустой массив при поиске — текст «Ничего не найдено», при первой загрузке — «Нет данных».»

**R — Reflective (рефлексивно)**

- Самопроверка:
- «Убедись: при смене страницы поисковой запрос не сбрасывается; при новом поиске пагинация сбрасывается на первую страницу; повторный запрос при том же q и page не уходит без необходимости.»
