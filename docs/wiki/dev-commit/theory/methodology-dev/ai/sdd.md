# Spec-Driven Development (SDD)

## Информация

::: tip Spec-Driven Development

- **Spec-Driven Development (SDD)** - это подход, когда вначале описывается спецификация, а потом LLM генерирует код на основе этой спецификации.
- Методология разработки программного обеспечения, при которой спецификации становятся источником истины вместо кода. Перед написанием реализации команда (или ассистент на базе ИИ) определяет, что должно делать система: требования, ожидаемое поведение, ограничения, критерии приёмки.
- В отличие от подходов «код сначала», где реализация предшествует документации, SDD требует, чтобы детальные спецификации (например, контракты API, планы архитектуры, критерии приёмки) создавались, проверялись и утверждались перед написанием первой строки производственного кода.

  :::

## Принципы

- **Детализированные спецификации**. В документе Software Design (SDD) описываются, что должно делать программное обеспечение, включая ожидаемые функции и важные факторы (безопасность, производительность). Часто используются визуальные средства и примеры, чтобы прояснить, как всё должно работать.
- **Ранний обзор**. Перед началом кодирования спецификации проверяют члены команды, тестировщики и иногда автоматические инструменты, чтобы выявить и исправить потенциальные проблемы.
- **Автоматизированная разработка**. Разработчики используют специальные инструменты, которые помогают превращать детальные планы непосредственно в рабочий код.
- **Постоянная поддержка**. После запуска программного обеспечения спецификации остаются полезным ориентиром для будущего сопровождения и обновлений.

## Структура

> Идея → Спецификация → План реализации → Декомпозиция задач → Код → Проверки
