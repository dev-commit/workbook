# Conditional Rendering

## Информация

::: tip Conditional Rendering

- **Conditional Rendering** (условная отрисовка) включает `v-show`, `v-if`, `v-else`, `v-else-if`
  :::

## Сравнения `v-if` и `v-show`

### `v-if`

::: details Описание

- Производит «настоящую» условную отрисовку, удостоверяясь что подписчики событий и дочерние компоненты внутри блока должным образом уничтожаются и воссоздаются при изменении истинности управляющего условия
- v-if ленивый: если условие ложно на момент первоначальной отрисовки, он не произведёт никаких действий — условный блок не будет отображён, пока условие не станет истинным
- Работает с псевдоэлементом `<template>`
- Совместное использование `v-if` и `v-for` не рекомендуется. При совместном использовании `v-if` и `v-for`, директива `v-for` имеет более высокий приоритет

:::

> Для `v-if` выше затраты на переключения, а у `v-show` больше затрат на первичную отрисовку. Если вы предполагаете, что переключения будут частыми, используйте `v-show`, если же редкими или маловероятными - `v-if`

```vue
<!-- В DOM дерево удаляется/добавляется узел  --->
<p v-if="view">text</p>
```

### `v-show`

- Не работает с псевдоэлементом `<template>` и с `v-else`

```vue
<!-- Элементу устанавливается display: none/block --->
<p v-show="view">text</p>
```

## Условная отрисовка

### `v-else`

- Вызывается при невыполнении if

```vue
<p v-if="view">text</p>
<p v-else>text</p>
```

### `v-else-if`

```vue
<div v-if="type === 'A'">A</div>
<div v-else-if="type === 'B'">B</div>
<div v-else-if="type === 'C'">C</div>
<div v-else>Не A/B/C</div>
```
